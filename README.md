# LuckyOx-SpringFramework

Spring 是每一位 Java 开发工程师都绕不开的话题。说得极端一些，如果只是会使用 Spring，却不了解它的核心原理和设计思想，就很难真正成长为一名高级 Java 工程师。

那我是励志想成为一名精通 Web 开发的信奥讲师，理解 Spring 框架的底层原理迫在眉睫。然而，对于我这种从接触 Java Web 开发开始，就一直在使用 Spring 框架的人来说，想要真正理解 Spring，很难很难。

因为在日常开发中，已经习惯了使用注解完成依赖注入，使用 Spring MVC 接收请求，使用 Spring Boot 自动完成各种配置。很多功能似乎理所当然地就能够运行，然而 Spring 在整个 Web 应用的启动和运行过程中，究竟替我们完成了哪些工作？如果没有 Spring，这些功能应该如何实现？这些我是不知道的，也很难从他人的知识库中理解这些。

正如苏轼在《题西林壁》中所写：

> 不识庐山真面目，只缘身在此山中。

因为从一开始就身处 Spring 构建的开发体系中，我们往往只知道如何使用它，却很难看清它的完整面貌。

因此，我打算从零开始，尝试复现一个简单的 Spring 框架，希望从实践中逐渐建立对 Spring 核心原理的整体认识。

本项目知识库：[LuckyOx-SpringFramework 知识库](https://www.yuque.com/luckyox/project/lu0cfhy94kc7u6vg)

参考文档：[廖雪峰老师的手写 Spring 博客](https://liaoxuefeng.com/books/summerframework/introduction/index.html)
